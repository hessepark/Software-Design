본 App은 GUI로 구성되어 있으며, 하드웨어적인 요소가 많이 들어간 아이디어이기 때문에
하드웨어적인 요소들은 충족했다고 가정합니다.
사용자 입장에서 제 아이디어가 들어간 HelmetSafer App이 어떠한 방식으로 돌아가는지 알 수 있게 Demo 형식으로
만들어보았습니다. 이클립스 환경이 없는 PC에서도 실행시키기 위해 JRE 파일을 EXE 파일과 함께 첨부했습니다. 하지만 콘솔창에 출력되는 내용도 있기 때문에
가급적이면 이클립스 환경에서 실행시키는 것을 권장합니다. (프로젝트 파일 하나 생성 후, Helmet_Safer안의 모든 파일들 넣어서 실행)

헬멧 코드와 핀 번호를 입력하는 창에서 키패드 기준 0 왼쪽이 Register(R)이고, 0 기준
오른쪽이 Delete(D)입니다. 다른 pc에서 실행 시 글씨가 깨지는 현상이 있어 간단한 알파벳으로
수정했습니다. 코드 입력과 설정 화면에서는 EXIT 버튼으로 로그인 창으로 갈 수 있습니다.

Analysis 단계에 있는 User Interface prototype과 거의 일치하게 구현하려고 노력했습니다.

* 해당 프로그램은 IntroScreen.java에 main 함수가 있어 IntroScreen.java를 선택한 후 실행시키시면 됩니다 :) *

1. 앱 실행 시, 3초간 화면이 유지되며 필요한 정보들을 불러옵니다.

2. Login 화면이 나타나면 Join 버튼을 눌러 로그인을 합니다. ID 중복 확인이 가능하며,
빈칸이 있는 채 회원가입 버튼을 누르게 되면 (please fill in all fields) pop up이 뜹니다.

3. 회원가입에 성공하게 되면 PIN number도 설정하겠냐는 팝업이 뜹니다. 아니오 버튼을 누르면
(You can now log in) 팝업이 뜨고 Login 화면으로 돌아갑니다.
아 경우, 회원가입 창에서 기입한 아이디와 비밀번호로 로그인을 하시면 됩니다. (아이디 비밀번호 불일치시 팝업 출력)
아이디와 비밀번호는 현재 User Manager라는 클래스를 통해 해쉬맵에 저장됩니다. (텍스트 파일에 저장하고 불러오려고 했으나 
보안 이슈가 있을 거 같아 후에 데이터베이스를 사용한 사용자 정보 저장 관리로 바꿔볼 예정)

현재는 Pin Number를 설정하고 싶으면 회원가입 후 예 버튼을 누르는 것이 필수입니다.

3.1 회원가입이 성공하게 되고, PIN number 설정에서 예를 누르면 PIN number 설정 창으로 이동합니다.
네 자리를 입력하지 않으면 팝업이 뜨고, 네 자리를 입력하면 PIN number가 등록됩니다.                                       (모든 코드들은 입력 후 R버튼 클릭, 한 글자씩 취소하고 싶을 경우 D버튼 클릭)
(콘솔 화면에서 가장 최근에 등록한 핀 번호 확인 가능->사용자는 본인이 입력한 PIN Number를 스스로
기억해야 하지만 하드웨어, 소프트웨어 적으로는 시스템에 전송을 해줘야 하므로 이렇게 표현)
이 경우, Login 창에서 PIN Number 버튼을 이용해 Pin Number로도 로그인이 가능합니다.

4. 6자리 Input Hcode 화면에서 임의의 6자리 Hcode를 입력합니다. 마찬가지로 콘솔에 등록한 Helmet
code가 출력됩니다.

5. 여러 조건들의 정보 확인 화면이 뜹니다. 빨간색 버튼은 정보 불러오기 실패를 했다는 동작이어서
다시 Hcode를 입력받는 화면으로 돌아가고 파란색 버튼은 정보 확인 성공이므로 
킥보드를 이용할 수 있게 됩니다. (모든 버튼들도 마찬가지로 클릭이 되었을 때 콘솔창에 표현이 된다)

6. 킥보드 이용 중 헬멧과 킥보드 사이의 거리가 일정 거리 이상으로 멀어진 것을 표현하기 위해 노란색 버튼을 두었습니다.
노란색 버튼 클릭 시 거리 멀어짐, 속도 제한이 걸립니다 화면을 출력하게 됩니다.

여기서 빨간색 버튼은 일정 거리의 기준치를 초과한 것으로 간주하고,  ( ex) 킥보드와 헬멧 사이의 거리가 10m 이하일 때는 속도제한만 걸리지만 30m를 벗어나게 되면) 곧 킥보드 작동을 멈추겠다는 문구를 띄우고
비상작동 버튼을 활성화시킵니다. 비상 작동 버튼을 누르게 되면 3분에서 시작해서 남은 시간이 표시됩니다. (현재는 4초로 설정)

7. 사용자가 킥보드 사용을 종료하면 Report 버튼이나 Warning 버튼을 눌러 필요한 정보를 매니저에게 전달하거나, 이번 사용 건에 대한 경고 내역을 확인할 수 있습니다
 초록색 버튼을 누르면 가장 초기 화면으로 돌아갑니다. Report 버튼을 누르고 요청 사항을 Submit 한 후, Home 버튼으로 돌아가는 것을 추천
